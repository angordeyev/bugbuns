# Decisions

## Commit Messages

Use present simple for sentences and do not put "." at the end, at least there is no multiple sentences.

## Branch Naming

Same name as feature or bug fix using kebab-case, examples: `some-feature`, `fix-docs-typos`.

## Unstage

    git reset

## Workflows

### Work on Branch

1. Make changes
2. Run `lg` (`lazygit`)
3. Review the changes and select files with <kbd>Space</kbd>
4. Press <kbd>c</kbd> and enter a the commit message
5. Push changes <kbd>P</kbd>
